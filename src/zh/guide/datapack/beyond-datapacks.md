# 比数据包更强大

完成第一次迁移后，Katton 就不只是“更舒服地调用命令”。它让你在保留快速编辑-重载循环的同时，使用接近模组级别的功能。

## 用事件代替轮询

数据包经常需要轮询，因为它不能直接订阅所有游戏事件。Katton 可以响应玩家加入、伤害、方块变化、战利品生成、聊天、世界 tick 等事件。

事件列表见 [事件](../events.md)。

## 注册表

在 Fabric 和 NeoForge 上，Katton 可以从脚本注册自定义内容：

- 物品；
- 方块；
- 实体；
- 效果；
- 粒子；
- 声音；
- 数据组件；
- 创造模式标签页。

可以从 [注册表概览](../registry/) 和 [实体教程](../registry/entity.md) 开始。

> [!NOTE]
> Paper 不支持注册表 API，因为 Paper 服务端无法控制原版客户端的模组注册表。

## 客户端功能

Fabric 和 NeoForge 脚本包可以包含客户端入口，用来编写 HUD、世界渲染、后处理效果和脚本包 UI 工作流。

见 [渲染](../render/) 和 [脚本包界面](../pack-ui.md)。

## 运行时注入

当事件和公开 API 不够用时，Katton 可以注入运行时方法。这是高级工具：很强，但比事件和普通 API 更容易误用。

见 [代码注入](../injection.md)。

## 迁移收益

迁移的最佳理由不只是“能力更强”。更重要的是代码更简单：更少命令字符串、更少伪变量、更少递归 schedule，更多直接访问你真正想修改的游戏对象。
